# 📊 SentinelX — Monitoring Guide

Basic monitoring setup using Prometheus and Grafana.

---

## Overview

| Tool | Purpose | URL |
|------|---------|-----|
| **Prometheus** | Collects metrics from the backend every 15 seconds | http://localhost:9090 |
| **Grafana** | Displays metrics as visual dashboards | http://localhost:3000 |
| **Backend /metrics** | Exposes application metrics in Prometheus format | http://localhost:8003/metrics |

---

## How It Works

```
SentinelX Backend ──── /metrics endpoint ────► Prometheus ────► Grafana
  (FastAPI app)         (auto-generated)       (collects        (displays
                                                every 15s)       dashboards)
```

1. The FastAPI backend automatically exposes a `/metrics` endpoint with data like:
   - Total number of HTTP requests
   - Request duration (how fast the API responds)
   - HTTP status codes (200 OK, 404 Not Found, 500 Error)
   - Requests broken down by endpoint

2. **Prometheus** scrapes this `/metrics` endpoint every 15 seconds and stores the data.

3. **Grafana** connects to Prometheus and lets you build dashboards to visualize the data.

## Quick Start

```bash
# Start everything including monitoring
docker compose up --build -d

# Check all services are running
docker compose ps
```

---

## Accessing the Tools

### Prometheus (http://localhost:9090)

1. Open http://localhost:9090
2. Go to **Status → Targets** to verify the backend is being monitored
3. Try a query in the search bar: `http_requests_total`

### Grafana (http://localhost:3000)

1. Open http://localhost:3000
2. Login with `admin` / `admin`
3. Skip the password change (or set a new one)
4. **Add Prometheus as a data source:**
   - Go to ⚙️ Settings → Data Sources → Add data source
   - Select **Prometheus**
   - Set URL to: `http://prometheus:9090`
   - Click **Save & Test**
5. **Create a dashboard:**
   - Go to ➕ → Dashboard → Add visualization
   - Select Prometheus as the data source
   - Use queries like:
     - `http_requests_total` — total API requests
     - `http_request_duration_seconds_bucket` — response times

## Metrics Available

| Metric | Description |
|--------|-------------|
| `http_requests_total` | Total number of HTTP requests |
| `http_request_duration_seconds` | How long each request takes |
| `http_request_size_bytes` | Size of incoming requests |
| `http_response_size_bytes` | Size of outgoing responses |
| `http_requests_in_progress` | Currently active requests |

| File | Purpose |

| `backend/main.py` | `/metrics` endpoint (auto-generated by prometheus-fastapi-instrumentator) |

## Troubleshooting

| Problem | Solution |
|---------|----------|
| Prometheus target shows "DOWN" | Make sure backend is healthy: `docker compose ps` |
| Grafana can't connect to Prometheus | Use `http://prometheus:9090` (Docker DNS), not `localhost` |
| No metrics showing | Hit some API endpoints first to generate data: `curl http://localhost:8003/health` |
